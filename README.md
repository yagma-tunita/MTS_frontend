# MTS 前端 - 航运物流管理系统

基于 Vue 3 + Element Plus + Vite 的航运物流管理前端。

## 技术栈

- **框架**: Vue 3.5+ (Composition API + `<script setup>`)
- **UI 组件库**: Element Plus 2.9+
- **构建工具**: Vite 6.3+
- **HTTP 客户端**: Axios
- **路由**: Vue Router 4
- **角色布局**: 货主(shipper) / 海运公司(shipping) / 管理员(admin)

## 快速启动

```bash
cd frontend
npm install
npm run dev
```

开发服务器默认运行在 `http://localhost:5173`，API 请求通过 Vite proxy 转发到 `http://localhost:8080`。

## 角色功能对照

### 货主 (shipper)

| 页面 | 功能 |
|------|------|
| 工作台 | 统计概览 + 最近订单 |
| 创建订单 | 5 步向导：选港 → 航线推荐 → 货物信息 → 联系方式 → 确认 |
| 订单管理 | 列表 + 详情（预计/实际航次双栏） + 取消 |
| 订单详情 | 订单信息 + 船舶信息 + 货物状态 + 时间线 + 支付 |
| 物流追踪 | 左侧订单列表 + 右侧追踪详情（船舶信息/货物状态/航次分栏） |
| 港口列表 | 查看港口信息 |
| 船舶列表 | 查看船舶详情 |
| 航线列表 | 查看航线及港口序列 |

### 海运公司 (shipping)

| 页面 | 功能 |
|------|------|
| 工作台 | 船舶/航线统计 + 最近订单 + 最近航次 |
| 订单管理 | 列表 + 细节 + **状态推进**（确认→发货→港口更新→完成） |
| 航线申请 | 创建新航线（选线/船/日期/途经港/单价） |
| 航次管理 | 查看靠泊记录 + 更新实际到/离港时间 |
| 报表 | 订单统计 + 航次利用率 |
| 基础数据 | 查看港口/船舶/航线 |

**订单状态操作：**
- **待确认 → 已确认**：确认接单，可填备注
- **已确认 → 运输中**：选港 + 实际离港时间 + **装货操作**
- **运输中 → 港口更新**：选港 + 到/离港时间 + **装/卸货操作**（可多次）
- **运输中 → 已完成**：选港 + 实际到港时间 + **卸货操作**

## API 约定

- **基础路径**: `/api/v1`
- **认证**: `Authorization: Bearer <access_token>`
- **响应格式**: `{ code: 0, message: "success", data: ..., meta: { page, page_size, total, total_pages } }`
- **Token 机制**: 15 分钟 access_token + 7 天 refresh_token
- **WebSocket**: `ws://localhost:8080/ws?token=<access_token>`

## 状态码

| code | 含义 |
|------|------|
| 0 | 成功 |
| 1000 | 请求参数错误 |
| 1001 | 认证失败 |
| 1002 | Token 过期 |
| 1003 | 无权限 |
| 1004 | 资源不存在 |

## 订单状态

| 值 | 状态 | 说明 |
|----|------|------|
| 0 | 待确认 | 货主已创建，等待船公司确认 |
| 1 | 已确认 | 船公司已接单 |
| 2 | 运输中 | 货物已装船，在途运输 |
| 3 | 已完成 | 货物已送达 |
| 4 | 已取消 | 订单取消 |

## 航线状态

| 值 | 状态 | 说明 |
|----|------|------|
| 0 | 待审核 | 海运公司申请，待管理员审批 |
| 1 | 已启用 | 审核通过，正常使用 |
| 2 | 已弃用 | 停止服务，可重新申请恢复 |

## 关键设计

- **三个角色共用同一套 API**，通过 JWT 中的 `role` 字段区分权限
- **数据隔离**：海运公司只能看到自己公司的订单和航线（通过 `load_note_id → voyage_cargo_note → shipping_line` 链路）
- **实际航次数据全部由人工录入**，系统不自动生成任何实际时间
- **货物装载状态**：按港口累计 LOAD/UNLOAD 操作，计算已装船/已卸货/待装船
- **船舶当前载货量** = 总装货 - 总卸货
